Background and introduction to Frame semantics

From Wikipedia:

Frame semantics is a theory of linguistic meaning that extends Charles J. Fillmore's, Fillmore (1976),case grammar. It relates linguistic semantics to encyclopaedic knowledge. The basic idea is that one cannot understand the meaning of a single word without access to all the essential knowledge that relates to that word. For example, one would not be able to understand the word "sell" without knowing anything about the situation of commercial transfer, which also involves, among other things, a seller, a buyer, goods, money, the relation between the money and the goods, the relations between the seller and the goods and the money, the relation between the buyer and the goods and the money and so on.

Thus, a word activates, or evokes, a frame of semantic knowledge relating to the specific concept it refers to (or highlights, in frame semantic terminology).

A semantic frame is a collection of facts that specify "characteristic features, attributes, and functions of a denotatum, and its characteristic interactions with things necessarily or typically associated with it." A semantic frame can also be defined as a coherent structure of related concepts that are related such that without knowledge of all of them, one does not have complete knowledge of any one; they are in that sense types of gestalt. Frames are based on recurring experiences. So the commercial transaction frame is based on recurring experiences of commercial transactions.

Words not only highlight individual concepts, but also specify a certain perspective from which the frame is viewed. For example "sell" views the situation from the perspective of the seller and "buy" from the perspective of the buyer. This, according to Fillmore, explains the observed asymmetries in many lexical relations.

FrameNet

The FrameNet project's goal is to build a database for semantic frames, their elements and associated sentences.

The original FrameNet database was created by hand: frames were defined along with their elements, lexical units were added to those elements and finally sentences were associated with the lexical units. The English FrameNet is constantly growing and now contains over a 1000 frames. The database is available for public use and can be used for various NLP tasks.

While the FrameNet annotated sentences are language specific, the frames themselves: their elements and hierarchy, are universal and can be used as a base for creating a similar database for languages other than the original English database and German, Japanese, Spanish and Danish FrameNets were created by doing just that.

The Hebrew FrameNet migration system

The first step of creating the Hebrew FrameNet was importing the frame hierarchy form the English FrameNet, as the frame structure is universal.

Next, lexical units, their sentences and the sentences' lexical annotation, all of which are language specific, must be specified. This task is not a trivial one and must be done manually.

Our project goal was creating a system which will assist inserting this data.

System main requirements:

  • Easy to use GUI which will allow quick insertation of lexical units, sentence association and sentance annotation
  • Intergarion with external tools such as automatic translation of English lexical units and automatic sentences searching systems
  • Users must be able to work simpultanously
  • Added content should be able to be reviewed and if neccessery corrected before inserated to the production DB
  • Expose an API for public use of the data gathered
